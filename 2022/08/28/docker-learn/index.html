What is Docker?

Docker is an open source application container , which can package programs and program environments together. In this way, it’s can fixed the problem of the unvailiability of the program due to eviromenttal changes. In a simple way to descript of it, it’s similar to virtual machine. Docker is smaller and faster than virtual machine, And the Docker does not virtualize the hardware. For using docker, the release of applications is also faster.

three parts of dock

  • images: images
  • container: container
  • reponsitory

Common commends

systemctl start/stop/status/enable docker

Respectively represent: start stop to check the status And startup docker

common image commends

docker images show the the images in the local system , can use -a/-q to show all or only show images id

docker search [option] name find the images from https://hub.docker.com options:

docker pull name[:TAG] download the images

docker system df check the space occupied by images or container or data

docker rmi name/image_id delete the image which named name,use -f to force deletion,we also can use this to delete some images in once; it also supports parameter passing ,you can delete all by this:docker rmi $(docker images -qa) it’s similar to shell rm commend

[^tips]: responsitory name, tags are < none > images commonly known as dangling images

common container commends

new and start a container:

docker run [OPTIONS] IMGAE [COMMAND] [ARG..]

common options:

  • –name set the container name
  • -d run the container in the background and return the container ID
  • -i run the container in an interactive mode, usually used with -t
  • -t Start the interactive container (there is a pseudo -terminal at the front desk, waiting for interaction)
  • -P random port mapping
  • -p set the port mapping

docker ps show all containers that are running

exit the container

  • exit run the container and exit ,the container stoped
  • ctrl +p+q run the container and ctrl+p+q ,the container do not stop

docker start container’s id/container’s name start the container which is already exit and stoped container

docker restart container’s id/container’s name

docker stopcontainer’s id/container’s name

docker kill container’s id/container’s name

delete the container:

docker rm container’s id/container’s name

ways to get into the container which was running

there two ways to get into it

  • ​ docker exec -it container_id bash shell
  • ​ docker attach container_id

both commands can enter the running container interaction mode,but there are some differences between them.

attach is just enter the container terminal, do not create a another process, if you used this way to enter,then once you using command exit,it will lead the container stop.

exec is enter the container and it will open a new terminal,and can start a new precess,in this way, using the command exit do not cause the container stop.

In some instance containers that do not need to be run in the foreground, usually we’ll using docker run -d images to start and run the container in the background. then using the docker exec -it container_id/name bashshell to enter container to do some operation

copy file from container to localhost:

docker cp containerID:path_in_container path_localhost

export and import container

docker export container_id/name > *.tar

cat file_name.tar |docker import - container_name/images:version

images

images is a lightweight executable independetn software package, which contains all the contents required to run the software that we made. We package the application and configuration dependencies to from a deliverable running environment. This package was called images file.

UnionFS is the base of the Docker images.

All images is readonly, only the container is writable.

commit

docker commit is likes to git: submit a copy of the container and generate a new image

`docker commit -m=‘’description‘’ -a=‘’author‘’ ‘’container’’ create_images_name:[tags]

container

some tips:

If Docker mount host directory to access if “can not to open directory.: Permission denied” appears, we can use this way to fixed the problem:

Add --privileged=true to the end of the mount host directory.

Docker volume

Docker volume is an another file path or file, it can be used to more than one container, the volume is mount the the container by Docker, but it’s not belong to UFS(Union File System). It’ll provide some features for continuous storage or sharing of data.

It’s means to make the Docker container ‘s persist data, to keep. In my opinion, it’s more like to a backup to make sure the important data do not be deleted while the container was deleted.

mount command:

docekr run -it --privileged=true -v/宿主机绝对路径:/容器内目录:rw/ro imagesname

rw: read and write.

ro: read only, it’s only limits the containers.

Install APPs in Dokcer

  1. search images
    • docker search name
  2. download images to local
    • docker pull name
  3. check images
    • docker images name
  4. start images
    1. `docker run -it -p port_local:port_container
  5. stop container
  6. remove container

install mysql

docker pull mysql:8.0

docker run -d -p 3306:3306 --privileged=ture -v /myname/mysql/log:/var/log/mysql -v /myname/mysql/data:/var/lib/mysql/ -v /myname/mysql/conf: /etc/mysql/conf.d -e MYSQL_ROOT_PASSWORD=123456 --name mysql mysql:8.0

install redis

docker pull redis

create a path in host: makdir -p /app/redis as the redis volume and copy the redis conf file to the path
